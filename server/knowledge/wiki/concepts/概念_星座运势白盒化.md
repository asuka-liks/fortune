## 定义

星座运势"白盒化"是指：在不改变星座符号体系的前提下，通过设计手段去除传统星座内容中的神秘化/预测式语调，将其重塑为"温暖的心理支持 + 基于星座特质的情景化建议"。Fortune 的星座技能是这一理念的工程实现。

## 传统星座内容的问题

| 传统写法 | 问题 |
|----------|------|
| "木星进入你的第五宫，带来浪漫机会..." | 假装是客观天象预测 |
| "幸运颜色：紫色，幸运数字：7，幸运方位：东南" | 随机生成但包装成"占星结果" |
| "本周金牛座财运亨通，投资有回报..." | 绝对化预测，创造虚假确定性 |
| "整体运势★★★☆☆，爱情★★★☆☆..." | 分节罗列，像产品说明书 |

## Fortune 的白盒化方案

### 1. 反教科书语调

Prompt 第一条就是"不要分章节列条目，不要'整体运势''爱情运势''事业学业''财富运势''健康运势'逐条罗列"。

设计意图：打破用户对"星座内容"的格式预期——没有星级评分、没有分类栏目，用户就不会把输出当成"占星报告"来读。

### 2. 幸运项克制

Prompt 要求"幸运颜色/数字/方位这些最多提一个，不用全列"。

设计意图：幸运项是星座内容中最"神秘化"的部分——它们暗示存在一种基于星座的客观幸运规律。通过数量限制，把"玄学功能"弱化为"趣味点缀"。

### 3. 星座特质一句带过

Prompt 要求"星座特质一句带过——这个星座当前周期的大方向一句话概括就行，别展开讲星座性格"。

设计意图：用户不需要被告知"金牛座的人固执但可靠"——他们自己就是金牛座。把篇幅留给"这个周期你可能遇到什么"，而不是"你是什么样的人"。

### 4. 运势 → 建议

Prompt 要求"把运势当成建议而不是天气预报"。

这是最关键的白盒化转换：

| 传统框架 | 白盒框架 |
|----------|---------|
| "本周你的爱情运势..."（预测） | "这段时间在感情上可以试着..."（建议） |
| "财运亨通"（断言） | "财务上建议留意..."（提醒） |
| 天象 → 命运 | 星座特质 → 处境认知 → 行动提示 |

### 5. 周期适配

Prompt 要求"不同周期（日运/周运/月运/年运）自然调整详细程度，不需要明说"。

设计意图：日运是情绪提示，年运是趋势感知——让 AI 自然控制信息粒度，而非机械套用模板。

## AstrologyForm 的用户交互设计

### 12 星座中英文双值

```typescript
// 中文
{ value: '白羊座 (3.21-4.19)', label: '♈ 白羊' }

// 英文
{ value: 'Aries (3.21-4.19)', label: '♈ Aries' }
```

星座值附带日期范围——UI 即教育，用户不需要额外记忆星座日期。

### 生日自动匹配

用户输入生日 → 自动计算星座（含跨年区间处理，如摩羯座 12.22-1.19）→ 自动填充选择。降低了"不知道自己星座"用户的进入门槛。

## 与塔罗/八字的对比

星座是五种技能中**最少"占卜感"**的一个（没有随机抽牌、没有铜钱起卦），但也是最容易被包装成"神秘预测"的一个（传统星座运程善于使用天文学术语制造权威感）。Fortune 的白盒化策略恰好反其道：**既然没有随机性，就干脆诚实做心理支持**。
