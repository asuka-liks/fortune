## 定义

AI 占卜 Prompt 设计模式是 Fortune 在五种占卜技能中反复验证过的 **LLM system prompt 编写方法论**——一套控制 AI 输出风格、结构、信息密度和价值观的可复用规则。

## 六条通用模式

### 1. 开篇人设一句定

每种 prompt 第一句就用一个角色定位句锚定语气：

| 技能 | 人设句 |
|------|--------|
| 八字 | "你是一位精通八字命理的大师，说话接地气，不拽文" |
| 星座 | "你是一位资深的西方占星师，说话温暖随意，不端架子" |
| 塔罗 | "你是一位经验丰富的塔罗牌解读师，风格像朋友聊天，不念教科书" |
| 八卦 | "你是一位精通《易经》的易学高手，说话接地气，不拽古文" |
| 趣味 | "你是一个在直播间用三国杀卡牌假装算命的主播，风格搞笑不正经" |

**规律**：`身份 + 风格约束 + 禁止项`——三个要素一句话到位。

### 2. 反教科书结构

所有 prompt 都**明确禁止**分章节列条目：

> "不要'整体运势''爱情运势''事业学业''财富运势''健康运势'逐条罗列"
> "不要'本卦解析''变卦解析''综合论断''行动建议'逐条写"

**规律**：LLM 不加约束时倾向产出结构化的分节内容（看起来"专业"但不像聊天）。要的是**口语化的自然段落**，不是学术小论文。

### 3. 焦点控制：科普 → 用户

每种 prompt 都有明确的焦点指令：

- 八字："重点回到用户——围绕用户可能的当下处境展开聊"
- 塔罗："每张牌快速过（一两句）→ 重点串联到用户"
- 八卦："先说明卦象结构（一句话）→ 重点回到用户"
- 星座："星座特质一句带过 → 重点回到用户"

**规律**：`符号信息（简要） → 用户处境（重点）`——这是算命场景的核心节奏：用户不在乎"甲木的性格特征"，在乎"我的感情什么时候好转"。

### 4. 信息密度控制

每种 prompt 精确控制各部分的篇幅比例：

| 内容类型 | 允许的篇幅 | 禁止的行为 |
|----------|-----------|-----------|
| 符号科普 | 一句话 | "别展开讲图案意象和神话背景" |
| 排盘/卦象 | 简要列出 | "别展开讲五行生克、六亲关系（除非直接相关）" |
| 用户解读 | 主力输出 | — |
| 附赠信息 | 最多一个 | "幸运颜色/数字/方位最多提一个" |

### 5. 非宿命化价值观注入

每种 prompt 都有价值观层面的约束：

- 八字："保持正向，趋吉避凶，不搞宿命论"
- 塔罗："不搞绝对化预测，强调自由意志"
- 八卦："不搞绝对化预测，强调'天行健，君子以自强不息'"
- 趣味：（不适用，本身就是玩笑模式）

**规律**：AI 算命最危险的陷阱是用户当真——必须用 prompt 层面约束来防止 AI 输出"你注定会..."这类语言。

### 6. 统一免责声明

每种 prompt 结尾强制：

> "⚠️ 以上内容由 AI 生成，仅供娱乐参考，请理性看待。"

除趣味模式外，其余四种中英文版本均以此结尾。这是 [[overview/AI算命应用的设计哲学]] 中最核心的伦理设计——**诚实比神秘感更重要**。

## 趣味模式的特殊设计

趣味模式是**唯一一个反常规**的 prompt——主动打破前5条规则：

- 人设不是"大师"而是"搞笑主播"
- 允许"强行附会，越离谱越好笑"
- 要求展示计算过程（"X除以8余Y，对应Z卦"）
- 主播体口语（"好"、"OK"、"什么意思呢"、"好再见"）

这恰恰是趣味模式的建构论意义：**当 Prompt 主动暴露解读者（AI）和解读过程时，占卜的神秘感就被白盒化了**——用户看到的不再是"天意"，而是"这个人（AI）根据规则编了个故事"。

## 可复用的通用模板

基于以上6条模式，为任意新的占卜类型设计 prompt 的通用公式：

```
1. 人设句：你是一位 [身份]，风格 [形容词] [形容词]，不 [禁止行为]
2. 输入信息区：## 用户信息 / 抽牌结果 / ...
3. 结构反约束：禁止分章节、禁止列条目、禁止教科书腔
4. 焦点指令：符号信息简要 → 重点回到用户
5. 密度指令：每类信息的篇幅上限
6. 价值观指令：不搞宿命论 / 不搞绝对化 / 强调自由意志
7. 免责声明：⚠️ 以上内容由 AI 生成...
```
